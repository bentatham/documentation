Java

DataStoreFactory is used to get the instances of different datastores.

AdminTest.spec

Admin test
==========

Admin user
----------
* login with "root"
* user should be admin


Normal user
-----------
* login with "user123"
* user should not be admin

StepImplementation.java

    @Step("login with <username>")
    public void login(String userName) {
        //logic to login
        DataStoreFactory.getScenarioDataStore().put("user", userName);
    }

    @Step("user should not be admin")
    public void shouldNotBeAdmin() {
        String userName = (String) DataStoreFactory.getScenarioDataStore().get("user");
        //logic to check admin status
    }

    @Step("user should be admin")
    public void shouldBeAdmin() {
        String userName = (String) DataStoreFactory.getScenarioDataStore().get("user");
        //logic to check admin status
    }

In the above example, the user was stored in the scenarioDataStore. And since the scope is for a scenario, user value of "admin" is valid only for Admin user scenario. The subsequent scenario Normal user would not be able to access the old value, but it can then set its own value to be used in the scenario ("user123" in this case)

Similarly

        DataStore scenarioDataStore = DataStoreFactory.getScenarioDataStore();
        DataStore specDataStore = DataStoreFactory.getSpecDataStore();
        DataStore suiteDataStore = DataStoreFactory.getSuiteDataStore();

can be used to access different datastores
